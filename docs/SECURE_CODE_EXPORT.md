# Secure & Clean Source Code Export Guide

This document describes the standard procedure for exporting the Clean-Core.io codebase for compliance reviews, external audits (like Codex), or clean deployments. 

The goal of this process is to ensure the export is:
1. **Secure**: Zero cleartext API keys, environment variables, or private configuration files.
2. **Clean**: Excludes development scrapbooks, temporary scripts, and platform-specific lock files.
3. **Small**: Excludes heavy package dependencies (`node_modules`), build caches (`.next`), and local logs.

---

## 🚫 What to Exclude

Always omit the following categories from any source archive:

| Category | Patterns / Folders | Reason |
| :--- | :--- | :--- |
| **Secrets & Config** | `.env`, `.env.local`, `*.key`, `*.pem` | Avoid credential leaks (e.g. Firebase Admin keys, Gemini BYOK). |
| **Dependencies** | `node_modules/` | Can be regenerated; extremely large and redundant. (`package-lock.json` is included for dependency audit purposes). |
| **Build Artifacts** | `.next/`, `dist/`, `build/`, `*.tsbuildinfo` | Temporary output generated by compilation processes. |
| **Agent Scratchpads** | `.agents/`, `.antigravity/`, `scratch/` | Contains developer-agent logs, prompt history, and helper scripts. |
| **Test Output** | `playwright-report/`, `test-results/` | Temporary browser session traces and test screenshots. |
| **Temporary Files** | `*.log`, `*.bak`, `*.tmp` | Redundant operating system or editor backups. |

## 🛠️ Export Methods

Choose the appropriate method depending on your operating system.

### Method A: Automated Windows PowerShell Script (Recommended)

An automated utility script is located at [scripts/export-source.ps1](file:///c:/Users/felix/antigravity/Project-Platform/scripts/export-source.ps1). It implements a secure stage-and-copy process.

**How to run it:**
1. Open a PowerShell prompt.
2. Run the script:
   ```powershell
   powershell -ExecutionPolicy Bypass -File scripts/export-source.ps1
   ```
3. The script will clean up previous exports, stage only non-excluded files in a temporary directory, compress it, and output the archive directly to your Desktop as `clean-core-src.zip`.

---

### Method B: Git-Based Export (Cross-Platform / Git-native)

If you are using Git, you can leverage the built-in `git archive` command. This is highly secure because it automatically excludes any files tracked under `.gitignore` or not checked into the repository index.

To add custom exclusions (like `.agents` or `scratch` if they are tracked):
1. Add an `.gitattributes` rule in the root directory:
   ```text
   .agents/ export-ignore
   .antigravity/ export-ignore
   scratch/ export-ignore
   .env.local export-ignore
   .env export-ignore
   ```
2. Run the export command:
   ```bash
   git archive --format=zip HEAD -o ~/Desktop/clean-core-src.zip
   ```

## 🔒 Cryptographic Manifest & Signature Verification

To satisfy enterprise auditing requirements, the export process automatically generates a `manifest.json` in the root of the zipped archive.

### Manifest Content
The manifest contains:
- `exportTimestamp`: ISO timestamp of the export.
- `platformVersion`: Current package version of the platform (e.g. `v1.16.0`).
- `exclusionRules`: List of exclusion patterns applied.
- `filesCount`: Total number of exported files.
- `manifestHash`: SHA-256 hash of the canonical file mapping.
- `signed`: Boolean flag indicating if the manifest is signed.
- `signature`: HMAC-SHA256 signature generated over the canonical manifest using the `AUDIT_SIGNING_KEY` environment variable.
- `files`: Array of file entries, each containing `path`, `size`, and `sha256` digest.

### Verifying the Export Archive
You can verify the integrity and authenticity of any exported ZIP file using [scripts/verify-export.ps1](file:///c:/Users/felix/antigravity/Project-Platform/scripts/verify-export.ps1):

1. Run the verification script:
   ```powershell
   powershell -File scripts/verify-export.ps1 -ZipPath C:\Users\felix\OneDrive\Desktop\clean-core-src.zip
   ```
2. To verify the signature authenticity as well, set the signing key in the environment or pass it directly:
   ```powershell
   $env:AUDIT_SIGNING_KEY="your-secret-signing-key"
   powershell -File scripts/verify-export.ps1 -ZipPath C:\Users\felix\OneDrive\Desktop\clean-core-src.zip
   ```

The script will extract the archive, calculate file hashes, match them against the manifest, verify the manifest digest, and validate the cryptographic signature.

## 🔒 Post-Export Verification Checklist

Before sending the ZIP file to anyone:
1. **Size check**: The final ZIP should be small (usually `< 5 MB` without node_modules). If it is `> 50 MB`, it likely contains `node_modules` or `.next`.
2. **Signature verification**: Run the verification script to ensure the archive is complete and untampered.
3. **Secret Scan**: Run a quick grep check inside the extracted archive to ensure no secrets were accidentally staged:
   ```bash
   # Look for environment variable leftovers
   grep -rn "API_KEY" path/to/extracted/
   ```
